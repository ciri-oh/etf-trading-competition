# 巡检报告模板

> 巡检员 cron job 生成报告时参考此模板。发现异动时输出，无异常时绝对静默。

---

## 报告结构

```markdown
【巡检报告 YYYY-MM-DD HH:MM】

## 市场概况
- 上证指数：XXXX（±X.XX%）
- 深证成指：XXXX（±X.XX%）
- 创业板指：XXXX（±X.XX%）
- 市场特征：（一句话概括）

## 持仓表现
| 标的 | 代码 | 数量 | 成本 | 现价 | 今日涨幅 | 持仓占比 |
|------|------|------|------|------|----------|----------|
| ... | ... | ... | ... | ... | ... | ... |
| **合计** | - | - | - | - | **±X.XX%** | **XX.XX%** |

## 账户状态
- 总资产：XXX,XXX 元（±XX.XX%）
- 可用资金：XXX,XXX 元（XX.XX%）
- 当日盈亏：±X,XXX 元（±X.XX%）
- 累计盈亏：±XX,XXX 元

## 异动发现
1. **⚠️ 异动描述**：具体说明
2. ...

## 偏离预警
- 策略员决策：「...」
- 实际执行：...
- 偏离性质：（主动/被动/市场推高）
- 风险评估：...

## 建议
1. 具体建议
2. ...

---
巡检完成，（一句话总结）。
```

## 异动判定标准

以下任一条件触发巡检报告：

| 类型 | 触发条件 | 严重程度 |
|------|---------|---------|
| 仓位偏离 | 总仓位偏离策略目标 >10个百分点 | ⚠️ 中 |
| 单只上限 | 单只ETF仓位 >60%（接近硬上限） | ⚠️ 中 |
| 单只超限 | 单只ETF仓位 >60%（突破硬上限） | 🔴 高 |
| 大幅回撤 | 单日亏损 >3%（占总资产） | 🔴 高 |
| 持仓止损 | 单笔亏损≥3%（减半仓线） | 🔴 高 |
| 持仓止损 | 单笔亏损≥4%（清仓线） | 🔴 高 |
| 持仓止盈 | 单笔盈利≥10%（止损上移至3%盈利） | ⚠️ 中 |
| 持仓超限 | 单只仓位>60%（需减仓） | 🔴 高 |
| 板块异动 | 持仓板块单日涨跌幅 >5% | ⚠️ 中 |
| 策略偏离 | 交易员执行与策略指令不一致 | 🔴 高 |
| 市场极端 | 大盘单日涨跌 >3% | ⚠️ 中 |
| 流动性 | 可用资金 <15%（应对能力不足） | ⚠️ 中 |
| **收敛转跌** | 策略员说"涨幅收敛"但持仓翻绿转跌，且连续两个巡检周期跌幅扩大 | ⚠️ 中 |
| **尾盘加速** | 14:30后持仓跌幅较前一巡检周期扩大>1.5% | ⚠️ 中 |
| **建仓后走弱** | 策略新建仓标的当日即转跌，但止损线未触发 | ℹ️ 低（仅记录，不触发异动） |
| **ETF与板块指数背离** | 板块指数反弹但持仓ETF不跟涨反跌，且背离幅度>2个百分点 | 🔴 高 |
| **条件满足后继续走弱** | 策略触发条件（价格≤目标价）执行后，价格继续下跌超过目标价区间 | ⚠️ 中 |
| **反弹衰竭** | 早盘大幅反弹（如创业板>1.5%）后午后显著回落（回吐>50%涨幅），且创业板/科创50回落幅度>上证 | ⚠️ 中 |
| **盈利保护位持续失守** | 策略员设置的盈利保护位在午后连续两个巡检周期（13:00-14:30）均低于保护位且无收复迹象 | ⚠️ 中 |
| **利好传导延迟确认** | 策略员预期利好催化（如WSTS/SEAJ双重行业利好）开盘被兑现盘压制全线低开，但盘中V型反转翻红确认利好正在传导。开盘低开看似"利好被无视"，但1小时内全部翻红说明市场在消化 | ℹ️ 低（仅记录趋势，非风险异动） |
| **持仓亏损逼近止损线** | 单笔亏损接近策略员设置的止损线（如距3%止损线<0.5个百分点），但尚未触发。需标记为关注点，告知策略员该标的已进入敏感区间 | ⚠️ 中 |
| **强弱分化** | 持仓标的中部分强势翻红（如半导体V型反转+1.40%），部分持续走弱（如证券ETF从-0.26%恶化至-2.41%），形成明显分化。说明非系统性风险，而是个别标的独立走弱 | ⚠️ 中 |
| **极端结构性行情** | 成长/科技指数（科创50/创业板指）与大盘指数（上证指数）走势严重背离，如科创50暴涨>4%而上证指数接近平盘或下跌。说明资金高度集中于单一赛道（如半导体/AI算力），市场呈"指数震荡、个股极致分化"格局。存量资金调仓博弈特征明显，非增量入场行情 | ⚠️ 中 |

## 无异动判定

以下情况**不触发**报告，绝对静默：

- 持仓涨跌幅在 ±2% 以内
- 仓位在策略目标区间内
- 无策略偏离
- 市场无极端波动
- 交易员执行与策略一致

## 典型异动案例

### 案例1：被动仓位漂移（2026-06-25）
- **现象**：策略目标仓位60%，实际67.94%，偏离+7.94个百分点
- **原因**：芯片板块连续两日大涨（+5%、+8%），持仓市值被动推高
- **性质**：市场上涨导致，非主动加仓，零交易
- **处理**：巡检报告触发 → 策略员评估是否减仓至目标区间
- **教训**：即使零交易，市场波动也会导致仓位偏离策略目标，巡检需关注被动漂移

### 案例2：ETF与板块指数背离（2026-07-02）
- **现象**：半导体板块指数(991334)从开盘-4.09%反弹至-1.77%（反弹+2.32%），但芯片ETF 512760从-3.94%恶化至-4.84%，不跟涨反跌
- **原因**：可能原因包括成分股调仓偏差（ETF跟踪的指数成分股与板块指数成分股不同）、大额赎回导致净值偏差、或该ETF跟踪的半导体细分方向与板块整体不同
- **性质**：结构性风险——不是整体市场问题，而是特定ETF与板块指数脱钩
- **处理**：巡检报告标记为高优先级异动，策略员需评估是否调整该ETF持仓
- **教训**：板块指数反弹不等于持仓ETF反弹。巡检员必须同时查板块指数（queryIndicator）和持仓ETF（getQuote），对比两者的涨跌幅方向。背离幅度>2个百分点即触发高优先级异动

### 案例4：反弹衰竭 + 盈利保护位持续失守（2026-07-03）
- **现象**：早盘大幅反弹（创业板+2.00%，科创50+0.94%），但午后显著回落（创业板+0.53%回吐3/4涨幅，科创50+0.14%几乎归零）。策略员设置的芯片ETF盈利保护位（159995@3.110, 512760@1.456）在早盘被跌破后整个下午未能收复。
- **原因**：半导体板块连续3日主力资金净流出（7/1 -435亿, 7/2 -739亿, 7/3 -247亿），反弹缺乏资金支撑。午后资金追高意愿弱，反弹动能衰竭。
- **性质**：结构性风险——不是单边下跌，而是反弹失败，属于"假反弹"模式
- **处理**：巡检报告标记为中等优先级异动，提示策略员关注盈利保护位持续失守。若次日开盘继续走弱，则反弹失败确认。
- **教训**：早盘反弹不等于趋势反转。午后14:00-14:30是验证反弹可持续性的关键窗口。创业板/科创50的回落幅度比上证更能反映真实市场情绪。盈利保护位被跌破本身不是异动，但持续失守（连续两个巡检周期）是趋势信号。

### 案例3：条件满足后继续走弱（2026-07-02）
- **现象**：策略员条件"证券ETF若跌至1.16以下可加仓5%"被触发（09:36成交@1.159），但加仓后价格继续下跌至1.149（-2.30%）
- **原因**：策略条件设置合理（1.16以下），但券商板块当日整体偏弱（-1.86%），加仓后继续下探
- **性质**：正常执行偏差——策略条件正确执行，但市场持续走弱
- **处理**：巡检员记录但仅标记为中等优先级。策略员在下一轮评估中决定是否补仓或等待企稳
- **教训**：策略条件触发不等于价格见底。加仓后继续走弱是正常现象，只要止损线未触发，按策略持有观察

## 策略获取（Fallback 策略）

巡检员读取策略时，按以下优先级查找：

1. **Comments 中 `[STRATEGY]` 标签**：`hermes kanban show --json <task_id>` → 遍历 comments 找 `[STRATEGY]`
2. **`latest_summary` 字段**：JSON 顶层字段（非 comments），由 `kanban complete --summary` 写入
3. **Comments 中关键词搜索**：搜索包含「策略/决策/操作计划」的最新评论

⚠️ 验证策略日期是否为当天，过期策略不可用。

### ⚠️ strategist-morning 处于 running 状态（无 [STRATEGY]）

**场景**：巡检员 09:25-09:30 运行时，strategist-morning 任务可能仍为 `running` 状态（策略员尚未完成），此时 `[STRATEGY]` 评论为空。

**正确行为**：
1. 不放弃巡检——继续采集市场数据（大盘指数、持仓行情、板块轮动）
2. 从当日 info-collector 的 `[INFO]` 评论获取盘前背景（持仓明细、盘前行情、关键信号）
3. 写 `[PATROL]` 报告时注明「策略员尚未发布今日 [STRATEGY]，暂无参照基准」
4. 市场异动仍需报告——策略未出不代表市场无异动
5. 策略发布后由后续巡检周期自动对齐

**典型输出**：
```
[!] 异动发现: 半导体/芯片板块开盘前集体重挫
...
[背景]
策略员尚未发布今日[STRATEGY], 暂无参照基准

[建议关注]
4. 策略员发布[STRATEGY]后及时对齐操作方向
```

## 注意事项

1. **queryIndicator 超时处理**：板块资金流向查询可能超时（>120s），此时跳过该数据点，不重试，用已有数据完成报告
2. **价格查询用 getQuote**：持仓ETF价格必须用 `getQuote --stock-code <code> --exchange <SH|SZ>`，不用 queryIndicator
3. **仓位计算基准**：所有百分比均为**占总资产比例**，不是个股涨跌幅
4. **区分"收敛"和"转跌"**：策略员说"涨幅收敛"时，巡检员需确认是涨幅收窄还是翻绿转跌。涨幅收窄=正常波动，翻绿转跌=趋势改变。详见 [references/afternoon-deterioration-pattern-2026-07-01.md](references/afternoon-deterioration-pattern-2026-07-01.md)
5. **区分"反弹"和"反弹衰竭"**：早盘大幅反弹（创业板>1.5%）后午后显著回落（回吐>50%涨幅）是反弹衰竭信号。创业板/科创50的回落幅度比上证更能反映真实市场情绪。详见 [references/afternoon-rally-exhaustion-pattern-2026-07-03.md](references/afternoon-rally-exhaustion-pattern-2026-07-03.md)
6. **连续两个巡检周期确认趋势**：一个周期的波动可能是噪音，两个周期跌幅扩大且未收窄说明趋势确立
7. **科创50是半导体持仓的领先信号**：科创50暴跌往往预示半导体板块进一步下跌，巡检员需关注
8. **建仓后走弱是正常现象**：只要止损线未触发，按策略持有观察，不因短期浮亏触发异动报告
9. **尾盘30分钟是加速窗口**：14:30-15:00是A股尾盘加速时段，巡检员需关注是否加速下探
10. **报告写入 Kanban**：用变量传参方式提交，避免 pipe 被安全扫描拦截且 stdin 不被读取：

```bash
# 先写入临时文件
cat > /tmp/patrol_report.txt << 'EOF'
[PATROL] 巡检简报...
EOF

# 用变量传参（位置参数，不从 stdin 读取）
TEXT=$(cat /tmp/patrol_report.txt)
hermes kanban comment <task_id> "$TEXT" --author inspector
```

❌ 不要用 pipe：`cat file | hermes kanban comment <id> -` 会报 `error: the following arguments are required: text`，因为 `hermes kanban comment` 不从 stdin 读取文本。
11. **板块数据必须标注代码**（2026-06-30 新增）：巡检报告中的板块涨跌幅必须标注板块代码（如 991334），且必须通过 `queryIndicator` 实际查询验证，不能凭印象或估算填写。报告中注明数据来源和查询时间。
12. **信息员报告必须写入 kanban 评论**（2026-06-30 新增）：信息员 cron prompt 必须明确要求将完整报告写入 kanban 评论（带 [INFO] 标签），不能只写 summary。验证方法：`hermes kanban show --json <info-collector-id>` 检查 comments 数组是否包含完整报告内容。
