## 根因分析
1. **prompt过于简略**：「你只执行策略员的指令」缺少执行对照表
2. **无价格校验硬约束**：没写「当前价>目标价→不追高」
3. **无指令清单白名单**：没写「只有策略员说的才做，其他不动」

## 修复方案
1. 补充完整执行纪律表：买入需≤目标价、持有倾向观望就不操作、无语指令则静默
2. 新增铁律：「当前价>目标价→不追高」和「说持有/观望→不操作」
3. 写入新prompt到jobs.json（绕过cronjob create长度限制）
4. 暂停→恢复cron job强制刷新

## 经验教训
- Cron prompt不能过于精简，核心执行规则必须写完整
- 用 jobs.json 直接编辑可绕过 cronjob create 的长度限制
- 执行纪律表比自然语言描述更可靠
