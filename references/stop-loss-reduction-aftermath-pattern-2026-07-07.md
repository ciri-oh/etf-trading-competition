# 止损减仓后剩余仓位继续恶化模式 (2026-07-07)

## 场景

策略员指令执行止损减半仓后，剩余仓位继续恶化（亏损扩大），但策略员未发出新的指令。

## 案例：512880 证券ETF (2026-07-07)

### 时间线

| 时间 | 事件 | 512880价格 | 亏损幅度 | 仓位 |
|------|------|-----------|---------|------|
| 11:40 午盘 | 策略员标记止损线1.130(亏损-3%) | 1.134 | -2.69% | 203,100股(20%) |
| 13:07 | 触发止损，减半仓卖出101,500股@1.13 | 1.130 | -3.21% | 101,600股(10%) |
| 13:25 巡检 | 剩余仓位继续走弱 | 1.131 | -5.99% | 101,600股(10%) |
| 13:35 交易员 | 无新[STRATEGY]指令，静默 | 1.128 | -6.24% | 101,600股(10%) |

### 交易员正确行为

1. **确认前序交易已完成**：listPendingOrders 返回空 -> 13:07的卖出委托已成交
2. **读取最新[STRATEGY]**：11:40午盘策略为"维持持有"，止损减半仓已执行完毕
3. **无新指令 -> 静默**：策略员未发出"清仓剩余"或"继续减仓"的新指令，不自行操作
4. **不"二次执行"止损**：策略员指令是"减半仓"，不是"清仓"。已执行后不再自行追加卖出

### 关键规则

策略员: "减半仓" -> 交易员执行 -> 剩余仓位继续恶化 -> 无新指令 -> 静默

- **交易员不自行扩大止损幅度**：策略员说"减半仓"就是减半仓，不是"减半仓后如果还跌就全清"
- **前序交易已执行不等于策略失效**：策略员的"维持持有"判断仍然适用于剩余仓位
- **区分"策略已执行完毕"和"策略需要更新"**：已执行的止损是过去时，剩余仓位按最新[STRATEGY]处理
- **巡检员的后续报告是信息参考，不是交易指令**：巡检员报告"剩余仓位继续走弱"是信息，交易员不据此自行操作

### 与"止损逼近但不触发"模式的关系

| 模式 | 触发条件 | 交易员动作 |
|------|---------|-----------|
| 止损逼近但不触发 | 亏损接近止损线但未触及 | 标记关注，等价格到位或策略员指令 |
| 止损触发+减半仓 | 亏损>=3%触发止损线 | 执行减半仓，写[TRADE] |
| 减仓后继续恶化 | 减半仓后剩余仓位亏损扩大 | 无新指令->静默，不自行追加卖出 |

### 巡检要点

- 止损减仓后，巡检员应继续监控剩余仓位走势
- 若剩余仓位继续恶化，巡检员在[PATROL]中如实报告即可
- 是否需要进一步减仓由策略员在下一评估节点(11:40/14:40)决定
- 交易员不因"亏损扩大"而自行操作——策略员指令已执行完毕

### 验证方法

交易员在检查是否需要操作时，应依次验证：
1. listPendingOrders -> 确认前序委托已成交（空列表或仅新委托）
2. 读取最新[STRATEGY]评论 -> 确认是否有新指令
3. 检查[TRADE]评论 -> 确认前序交易是否已执行（避免重复执行同一指令）
4. 无新指令 -> 静默
